Detailed Description

Method settings for a model to be put through Bayesian updating.

Field Documentation

apop_model* apop_mcmc_settings::base_model

The model you provided with a log_likelihood or p element (which need not sum to one). You do not have to set this: if it is NULL on input to apop_model_metropolis, I will fill it in.

double apop_mcmc_settings::burnin

What percentage of the periods should be ignored as initialization. That is, this is a number between zero and one.

int apop_mcmc_settings::histosegments

If outputting a binned PMF, how many segments should it have?

double apop_mcmc_settings::last_ll

If you have already run mcmc, the last log likelihood in the chain.

long int apop_mcmc_settings::periods

For how many steps should the MCMC chain run?

apop_model* apop_mcmc_settings::pmf

If you have already run mcmc, I keep a pointer to the model so far here. Use apop_model_metropolis_draw to get one more draw.

apop_model* apop_mcmc_settings::proposal

The distribution from which test parameters will be drawn. If NULL, apop_model_metropolis will use a Multivariate Normal with the appropriate dimension, mean zero, and covariance matrix I. If not NULL, be sure to parameterize your model with an initial position.

int apop_mcmc_settings::proposal_is_cp

For internal use.

apop_data* apop_mcmc_settings::starting_pt

Deprecated and ignored. Starting point is drawn from your proposal distribution.

void(* apop_mcmc_settings::step_fn)(apop_data *, apop_model *)

Modifies the parameters of the proposal distribution given a successful draw. Thus, this function writes the drawn data point to the parameter set. If the draw is a scalar, the default function sets the 0th element of the model's parameter set with the draw (works for the apop_normal and other models). If the draw has multiple dimensions, they are all copied to the parameter set, which must have the same size.
